# Workstream 2: UX / UI

## Goal
Design a clean, low-friction copilot experience embedded in support workflows.

## Experience Principles
- Fast comprehension: show top suggestion first.
- Progressive disclosure: details on demand.
- Trust by design: source citations and confidence visible.

## Core Screens
- Copilot side panel in ticket view.
- Suggestion cards (reply draft, action plan, KB snippets).
- Review mode with editable response and citation drawer.
- Feedback controls (thumbs up/down + reason tags).

## Interaction Notes
- Keyboard-first actions for agents handling high ticket volume.
- State indicators: loading, low confidence, no-answer fallback.
- Accessibility baseline: WCAG AA contrast, focus states, screen reader labels.

## Deliverables
- Wireframes for panel and card system.
- Component inventory and interaction states.
- UI copy guidelines for confidence, risk, and escalation.

## Open Questions
- Should confidence be numeric or tiered labels (High/Medium/Low)?
- Should citations open inline or in modal for speed?
